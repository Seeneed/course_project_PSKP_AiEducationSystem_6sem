Курсовой проект защищен на оценку 9.

Приложение поддерживает четыре роли: администратор, преподаватель, обучающийся и гость.

Функции администратора:
1. Управление учетными записями.
2. Конфигурация параметров AI-моделей.

Функции преподавателя:
1. Загрузка исходных материалов (в виде PDF или DOCX файлов).
2. Настройка параметров генерации.
3. Редактирование сгенерированного контента.
4. Публикация учебных материалов.

Функции обучающегося:
1. Просмотр образовательных материалов.
2. Прохождение сгенерированных тестов.
3. Экспорт учебных материалов (в текстовый формат).

Функции гостя:
1. Регистрация.
2. Аутентификация.
3. Просмотр публичной информации.

Программное средство реализовано как web-приложение типа SPA на платформе Node.js с использованием языка JavaScript. Взаимодействие между клиентом и сервером осуществляется посредством REST/API на основе принципов асинхронного программирования. Архитектура приложения обеспечивает максимальную независимость представления, бизнес-логики и хранилища данных (Microsoft SQL Server) для возможности масштабирования.
Для обеспечения безопасности и управления статическими ресурсами был использован Proxy-сервер (Nginx) с поддержкой протоколов HTTPS/HTTP. Программное решение поддерживает контейнеризацию (Docker/Compose) для развертывания под разными платформами.
